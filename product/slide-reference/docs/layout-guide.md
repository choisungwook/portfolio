# 레이아웃 타입별 사용 가이드

## 왜 타입을 나누나

발표의 목적에 따라 최적의 레이아웃이 다르다. 공부 발표에서 C-Level 보고서 스타일을 쓰면 과하고, 임원 보고에서 코드 하이라이트 중심 레이아웃을 쓰면 핵심이 안 보인다. 타입별로 정해두면 AI agent가 자동으로 적합한 레이아웃을 선택할 수 있다.

## 타입 선택 가이드

| 발표 상황 | 권장 타입 | 이유 |
|-----------|-----------|------|
| 사내 스터디, 기술 블로그 발표 | study | 코드와 개념 설명에 최적화 |
| 밋업, 컨퍼런스 발표 | community | 청중의 시선을 끌어야 하므로 임팩트 있는 비주얼 |
| CTO/VP 보고, 이사회 발표 | executive | 결론 먼저, 수치 중심, 의사결정 유도 |
| 기술 선택, 마이그레이션 검토 | comparison | 두 가지를 나란히 비교하는 구조 |
| 로드맵, 장애 보고서, 프로세스 설명 | timeline | 시간 흐름이나 단계별 진행에 최적화 |

## 타입별 상세

### Study (공부용)

포함 슬라이드 패턴:
- **타이틀**: 그라데이션 배경, 중앙 정렬
- **개념 설명**: 2×2 그리드 카드
- **코드 하이라이트**: 코드 + 줄별 설명
- **핵심 요약**: 번호 매긴 키포인트

폰트 크기 범위: 16px-22px. 코드가 많아서 작은 폰트도 허용.

### Community (커뮤니티 발표용)

포함 슬라이드 패턴:
- **임팩트 타이틀**: 큰 텍스트, 그라데이션 하이라이트
- **문제 제기**: 큰 질문 하나로 청중 집중
- **아키텍처**: 노드 + 화살표 플로우
- **데모**: 터미널 UI 스타일
- **CTA**: 마무리 + 연락처

폰트 크기 범위: 20px-28px. 먼 거리에서도 읽혀야 한다.

### Executive (C-Level 발표용)

포함 슬라이드 패턴:
- **표지**: 미니멀, 상단 라인 강조
- **Executive Summary**: 결론 + 핵심 지표 3개
- **의사결정**: 옵션 비교 카드
- **로드맵**: 분기별 타임라인

폰트 크기 범위: 22px-30px. 임원은 먼 거리에서 보는 경우가 많다.

핵심 원칙: 결론을 먼저 보여주고, 근거를 뒤에 배치한다 (Pyramid Principle).

### Comparison (비교/분석용)

포함 슬라이드 패턴:
- **Before/After**: 좌우 분할, 색상 대비
- **비교 테이블**: 항목별 승자 하이라이트
- **장단점**: 2열 카드형
- **결론/판정**: 중앙 결론 박스

폰트 크기 범위: 18px-24px. 테이블이 있으므로 중간 크기.

### Timeline (타임라인/프로세스용)

포함 슬라이드 패턴:
- **수평 스텝**: 원형 번호 + 연결선
- **수직 타임라인**: 시간순 이벤트 (장애 보고서 스타일)
- **마이그레이션 단계**: 체크리스트 카드
- **플로우차트**: 의사결정 분기

폰트 크기 범위: 18px-24px.

## 반응형 폰트 설계 원칙

모든 레이아웃은 `--base-font-size` CSS 변수를 기준으로 상대 단위(`em`)를 사용한다. 이를 통해:

1. `--base-font-size`만 변경하면 전체 슬라이드의 크기 비율이 유지된다
2. 도형 안 텍스트는 `overflow: hidden`, `text-overflow: ellipsis`, `word-break: keep-all`로 넘침을 방지한다
3. 카드형 레이아웃은 `min-height: 0` + `overflow: auto`로 내용이 많아져도 스크롤로 대응한다

make-slide.vercel.app의 문제점이었던 "폰트 크기 변경 시 레이아웃 깨짐"을 이 방식으로 해결한다.
