# Slide Presentation Generator

HTML/CSS 기반 프레젠테이션 슬라이드를 생성한다. `product/slide-reference/layouts/`의 레이아웃 레퍼런스를 기반으로 발표 목적에 맞는 슬라이드를 만든다.

<design-principles>

이 스킬이 해결하는 핵심 문제는 두 가지다:

1. **폰트 가독성** — 폰트가 작으면 뒷자리에서 안 보이고, 크면 도형 안 텍스트가 넘친다. `--base-font-size` CSS 변수와 `em` 단위로 비율을 유지하면서 크기를 조절한다.
2. **도형 내 텍스트 배치** — 카드, 박스, 원형 도형 안의 텍스트는 `overflow: hidden`, `word-break: keep-all`, `text-overflow: ellipsis`로 넘침을 방지한다. 텍스트가 길면 줄이지, 도형을 깨뜨리지 않는다.

</design-principles>

<workflow>

1. 사용자의 발표 주제와 목적을 파악한다
2. 아래 레이아웃 타입 중 적합한 것을 선택하거나 사용자에게 제안한다
3. `product/slide-reference/layouts/` 디렉터리에서 해당 타입의 HTML을 읽고 구조를 참조한다
4. `product/slide-reference/docs/color-palettes.md`에서 팔레트를 선택한다
5. 레퍼런스의 구조와 스타일을 기반으로 슬라이드를 생성한다
6. 생성한 HTML을 사용자가 지정한 경로에 저장한다

</workflow>

<layout-types>

| 타입 | 레퍼런스 파일 | 용도 | 폰트 범위 |
|------|---------------|------|-----------|
| study | `layouts/study-default.html` | 개념 정리, 코드 하이라이트, 스터디 공유 | 16-22px |
| community | `layouts/community-default.html` | 밋업, 컨퍼런스, 기술 커뮤니티 | 20-28px |
| executive | `layouts/executive-default.html` | CTO/VP 보고, 이사회, 의사결정 | 22-30px |
| comparison | `layouts/comparison-default.html` | Before/After, 기술 비교, 장단점 | 18-24px |
| timeline | `layouts/timeline-default.html` | 로드맵, 장애 보고서, 프로세스 | 18-24px |

타입 선택 기준:
- 코드가 포함되면 → study
- 청중이 100명 이상이면 → community
- 보고 대상이 임원이면 → executive
- 두 가지를 비교하면 → comparison
- 시간 순서가 중요하면 → timeline

</layout-types>

<type-selection>

사용자가 타입을 지정하지 않으면, 발표 상황을 물어본다:

```
어떤 종류의 발표인가요?

1. 📖 공부/스터디 공유 (코드 중심)
2. 🎤 밋업/컨퍼런스 발표 (청중 많음)
3. 📊 임원 보고/의사결정 (결론 먼저)
4. ⚖️ 기술 비교/분석 (A vs B)
5. 📋 프로세스/타임라인 (단계별)
```

</type-selection>

<slide-generation-rules>

슬라이드 HTML 생성 시 지키는 규칙:

**구조**
- 슬라이드 컨테이너: `width: 960px; height: 540px` (16:9 비율)
- 모든 크기는 `em` 단위 사용 — `--base-font-size`만 바꾸면 전체 비율 유지
- 한 슬라이드당 하나의 메시지에 집중 — 3가지 이상 넣으면 어수선해진다

**타이포그래피**
- 제목: 1.4em-2.5em, font-weight 700-800
- 본문: 0.7em-0.9em, line-height 1.5 이상
- 코드: 0.6em-0.8em, monospace 폰트
- 한국어 텍스트는 `word-break: keep-all`로 어절 단위 줄바꿈

**도형 안 텍스트**
- 카드/박스: `padding: 1em` 이상, `overflow: hidden`
- 텍스트가 도형을 벗어나지 않도록 `min-height: 0` + `overflow: auto`
- 제목은 `flex-shrink: 0`으로 고정, 설명은 스크롤 또는 줄임

**색상**
- `product/slide-reference/docs/color-palettes.md`의 팔레트 사용
- CSS 변수로 정의하여 팔레트 교체가 쉽도록 구성

**빌드 도구 없음**
- 순수 HTML/CSS만 사용 — 브라우저에서 바로 열 수 있어야 한다
- 외부 CDN 의존성 없음 — 오프라인에서도 동작

</slide-generation-rules>

<output-format>

생성 결과는 단일 HTML 파일로 출력한다. 파일 상단에 메타 정보 주석을 포함한다:

```html
<!--
  Generated by: slide-presentation skill
  Type: {타입}
  Palette: {팔레트 이름}
  Slides: {슬라이드 수}
  Base font: {기본 폰트 크기}px
-->
```

</output-format>
