# python 작성 규칙

- indent는 2 spaces
- 프로젝트 관리, 패키지 관리, 파이썬 가상환경은 uv
- 코드를 성능관점에서 잘 짜는것 보다 잘 읽히는 코드를 작성
- 함수는 최대한 짧게 작성, 20라인 이상이면 리팩토링 고려
- 함수는 하나의 일만 하도록 작성
- 함수 이름은 snake_case로 작성
- 클래스 이름은 PascalCase로 작성
- 변수 이름은 snake_case로 작성
- 상수는 UPPER_SNAKE_CASE로 작성
- 타입 힌트 적극 활용
- docstring 작성, 함수의 목적과 입력/출력 설명
- 불필요한 주석은 피하고, 코드 자체가 명확하도록 작성
- 예외 처리는 명확하게, 필요한 경우 사용자 정의 예외 클래스 작성
