# 모니터링 및 디버깅 가이드

소켓 누수와 파일 디스크립터(FD) 고갈 문제를 추적하고 진단하기 위한 모니터링 및 쉘 디버깅 방법을 안내합니다.

## 1. Grafana를 이용한 시각적 모니터링

Grafana 대시보드는 애플리케이션의 리소스 누수 추이를 실시간으로 파악하는 데 유용합니다.

### 접속 방법

- URL: `http://localhost:3000` (ID/PW: admin/admin)
- 대시보드: **MariaDB Socket Leak Monitoring**

### 주요 모니터링 패널

1. **Process Open FDs (파일 디스크립터 수)**
   - **의미**: 애플리케이션 프로세스가 열고 있는 총 FD의 개수입니다.
   - **누수 판단**: API 호출 등 특정 이벤트 이후 이 수치가 계단식으로 지속 상승한다면 소켓(또는 파일) 누수를 강하게 의심해야 합니다.

2. **Zombie Connections (좀비 커넥션 수)**
   - **의미**: MariaDB 서버 관점에서 `SLEEP` 상태로 장시간 방치된 커넥션의 수입니다.
   - **누수 판단**: 애플리케이션은 이미 타임아웃으로 처리를 중단했는데 DB 쪽 커넥션 수가 떨어지지 않는다면, TCP 연결이 정상적으로 종료(`FIN`)되지 않은 것입니다.

3. **Process CPU Usage (CPU 사용률)**
   - **의미**: 애플리케이션의 CPU 사용량입니다.
   - **누수 판단**: 열린 소켓(FD)이 누적될수록 OS 커널이 소켓 상태를 감시하는 오버헤드가 커지므로, CPU 사용률이 점진적으로 상승하는 패턴을 보일 수 있습니다.

## 2. 쉘 환경에서의 디버깅 (CLI 도구)

컨테이너 또는 리눅스 서버에 직접 접속하여 OS 레벨의 지표를 확인하는 방법입니다.

### 1) 시스템 제한 확인 (`ulimit`)

프로세스가 최대로 열 수 있는 FD의 개수를 확인합니다. 누수가 발생하면 이 한계치에 도달하여 `Too many open files` 에러가 발생합니다.

```bash
# 앱 컨테이너 접속
docker exec -it socket-leak-app bash

# 현재 쉘/프로세스의 소프트(soft) 및 하드(hard) FD 제한 확인
ulimit -n
ulimit -Hn
```

### 2) 활성 소켓 및 네트워크 연결 확인 (`ss`, `netstat`)

실제 TCP 소켓이 어떤 상태에 머물러 있는지 확인합니다.

```bash
# 3306(MariaDB) 포트와 연결된 TCP 소켓 상태 출력
ss -tnp | grep 3306
```

- **`ESTABLISHED`**: 소켓이 정상 연결되어 통신 중이거나, 누수되어 **닫히지 않고 방치된 상태**입니다.
- **`FIN-WAIT-2` / `TIME-WAIT`**: 정상적인 소켓 종료 절차를 밟고 있는 상태입니다. (누수가 아님)
- **`CLOSE-WAIT`**: 상대방(DB)이 연결을 끊었으나, 내(App)가 아직 소켓을 닫지 않은 상태입니다. (애플리케이션 쪽 누수 징후)

### 3) 프로세스의 열린 파일(FD) 개별 확인 (`ls`, `lsof`)

리눅스에서 소켓은 파일 디스크립터로 취급되므로, 특정 프로세스가 열고 있는 FD 목록을 직접 조회할 수 있습니다.

```bash
# Java 프로세스의 PID 확인 (보통 컨테이너의 엔트리포인트면 PID 1)
ps -ef | grep java

# PID 1이 열고 있는 파일/소켓의 총 개수 확인
ls /proc/1/fd | wc -l

# 어떤 타입의 FD를 열고 있는지 상세 조회 (소켓 위주로 필터링)
lsof -p 1 | grep TCP
```

계속해서 `ls /proc/1/fd | wc -l` 수치가 증가한다면 누수가 확실합니다.

### 4) 실시간 리소스 모니터링 (`top`, `docker stats`)

누수로 인한 CPU/메모리 오버헤드 추이를 파악합니다.

```bash
# 컨테이너 외부(호스트)에서 전체 컨테이너 리소스 사용량 실시간 확인
docker stats

# 컨테이너 내부에서 Java 프로세스의 CPU/메모리 점유율 확인
top
```
