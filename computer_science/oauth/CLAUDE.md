# 프로젝트 목적

oauth와 oidc 원리를 공부합니다. 원리를 공부하고 핸즈온을 할겁니다. 원리를 공부하는 이유는 다른 프로젝트와 의사소통과 의사결정을 위해서입니다.

## 자기소개

저는 데브옵스 엔지니어이서 쿠버네티스, aws 등에 친숙합니다.

## 공부 스타일

이 프로젝트 또는 저의 공부 스타일은 핸즈온을 하면서 디버깅하며 이론을 확인하는 겁니다. 난이도는 입문 또는 초급, 중급 수준입니다.

## requiremnets

- 핸즈온은 가능한  제 pc에서 코드실행 또는  도커 컨테이너를 실행할거에요.
- 도커는 docker compose파일과 dockerfile, makefile을 선호합니다.
- oauth, oidc 주체들 예: service provider 등의 개념도 공부해야하고 필요하다면 github, google 같은 saas를 쓰고 싶어요.

## 코드 스타일

- 언어는 python을 선호하고 FastAPI가 편합니다. python 패키지 관리는 uv로 해주세요. uv로 python 가상환경도 사용합니다. python은 3.12를 사용할게요
- 프론트엔드가 필요하면 Reactjs 등 프레임워크없이 순수 javascript 좋을 것 같아요. 자바스크립트엔진은 nodejs를 사용합니다.
- 언어 indent는 2입니다.
- 테스트코드 (백엔드에 한함)도 간단히 작성해주세요.
- 코드 스타일은 코드를 잘 짜는게 목표가 아니라 가독성 있게 즉 학습을 할 수 있게 도와주는 수단입니다. 알고리즘대회처럼 코드를 잘 짤 필요가 없어요. 대신, 함수, 클래스 등을 잘 사용해서 읽기 좋은 코드여야 합니다.
